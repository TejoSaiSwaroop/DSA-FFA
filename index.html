Arrays

An array is a row of boxes numbered from 0. Each box holds one value. The box number is called an index. Arrays are the foundation of nearly every algorithm — everything builds on this.

Read/write O(1) Search O(n) Index starts at 0 Fixed size
What is an index?
Indexes start at 0. The first item is [0], the second is [1], the last is always [length − 1]. Click any box to explore.

💡Negative indexes count from the end. arr[-1] = last item, arr[-2] = second-to-last. Python supports this with no extra math.

Two SumEasy

Find two numbers that add up to a target. Return their indexes. Classic first problem — start with a brute-force O(n²) loop, then optimise with a dictionary.

Input: [2,7,11,15], target=9 Output: [0,1] ← 2+7=9
LeetCode ↗
For each number n at index i, the answer partner is (target - n). Check if that partner already exists in a dict. If yes → done! If no → store {n: i}.
def two_sum(nums, target):
    seen = {}
    for i, n in enumerate(nums):
        complement = target - n
        if complement in seen:
            return [seen[complement], i]
        seen[n] = i
    # O(n) time, O(n) space
Best time to buy and sell stockEasy

Find the maximum profit from one buy/sell. Track the minimum price seen so far and compute max profit at each step.

Input: [7,1,5,3,6,4] Output: 5 ← buy at 1, sell at 6
LeetCode ↗
One pass: track min_price seen so far. At each price, compute profit = price - min_price. Track max profit seen.
def max_profit(prices):
    min_p = prices[0]
    best = 0
    for p in prices:
        min_p = min(min_p, p)
        best = max(best, p - min_p)
    return best
Contains duplicateEasy

Return True if any value appears at least twice. Use a set for O(n) time — if you try to add a value already in the set, it's a duplicate.

Input: [1,2,3,1] → True Input: [1,2,3,4] → False
LeetCode ↗
seen = set(). For each n in nums: if n in seen → return True. Else seen.add(n). Return False if loop finishes.
def contains_duplicate(nums):
    seen = set()
    for n in nums:
        if n in seen: return True
        seen.add(n)
    return False

Longest common prefixEasy

Find the longest prefix string shared by all strings in an array. Compare character by character using the first string as a reference.

LeetCode ↗

def longest_prefix(strs):
    prefix = strs[0]
    for s in strs[1:]:
        while not s.startswith(prefix):
            prefix = prefix[:-1]
    return prefix

Lists (dynamic arrays)

Python lists can grow and shrink at any time. You can add, remove, sort, and rearrange freely. They're backed by a dynamic array internally — when they run out of space, Python automatically allocates a bigger one.

Dynamic size append O(1) insert/remove O(n) pop() O(1)

Operations with explanations
PYTHON
nums = [10, 20, 30]

nums.append(40)         # [10,20,30,40]     O(1) — fast
nums.insert(1, 99)      # [10,99,20,30,40]  O(n) — shifts!
nums.remove(99)         # removes first occurrence
nums.pop()               # removes & returns last   O(1)
nums.pop(0)             # removes at index 0       O(n)

len(nums)                # count elements
20 in nums               # True/False membership
nums.sort()              # sort in-place
sorted(nums)             # returns new sorted list
nums[::-1]               # reversed copy
3append(40) — adds to the end. O(1) amortised — occasionally Python needs to resize the internal array, but on average it's instant.
4insert(1,99) — inserts at index 1. O(n) because every element from index 1 onwards has to shift right by one position.
5remove(99) — scans left to right and removes the first element equal to 99. O(n) scan + O(n) shift = O(n) total.
6pop() — removes the last element and returns it. O(1) — no shifting needed. Use this, not pop(0), when you can.
7pop(0) — removes the first element. O(n) because everything else shifts left. For frequent front-removals, use a deque instead.
10nums.sort() — Timsort algorithm, O(n log n). Modifies in place, returns None. Use sorted(nums) to get a new list instead.

1from collections import deque — deque = double-ended queue. It's a specialised data structure optimised for O(1) adds/removes from both ends.
4q.append() — adds to the right (back). O(1). Same as a regular list's append.
8q[0] — peek at the front without removing it. O(1) with deque.

11while q: — a non-empty deque is truthy. This loop runs until the queue is completely empty.

Implement queue using stacksEasy

Build a queue using only two stacks — no deque allowed. Key insight: two reversals cancel out.

push(1),push(2),peek()→1,pop()→1
LeetCode ↗
Use two lists: inbox and outbox. Push to inbox. To pop/peek: if outbox empty, move all inbox items to outbox (this reverses them). Pop from outbox.
class MyQueue:
    def __init__(self): self.In=[]; self.Out=[]
    def push(self,x): self.In.append(x)
    def _move(self):
        if not self.Out:
            while self.In: self.Out.append(self.In.pop())
    def pop(self): self._move(); return self.Out.pop()
    def peek(self): self._move(); return self.Out[-1]
Number of islandsMedium

Count connected regions of '1's. BFS each island with a queue — mark cells visited as you explore.

[["1","1","0"],["0","1","0"],["0","0","1"]] → 2
LeetCode ↗
Loop every cell. Unvisited '1' found → count++, BFS. BFS: queue this cell, mark visited, pop and check all 4 neighbours.
def num_islands(grid):
    from collections import deque
    count = 0
    for r in range(len(grid)):
        for c in range(len(grid[0])):
            if grid[r][c] == "1":
                count += 1
                q = deque([(r,c)]); grid[r][c]="0"
                while q:
                    x,y = q.popleft()
                    for dx,dy in [(0,1),(0,-1),(1,0),(-1,0)]:
                        nx,ny=x+dx,y+dy
                        if 0<=nx<len(grid) and 0<=ny<len(grid[0]) and grid[nx][ny]=="1":
                            grid[nx][ny]="0"; q.append((nx,ny))
    return count

Min stackMedium

Build a stack that also tracks the minimum in O(1) at all times, even after pops.

class MinStack:
    def __init__(self): self.s=[]; self.m=[float('inf')]
    def push(self,x):
        self.s.append(x)
        self.m.append(min(x, self.m[-1]))
    def pop(self): self.s.pop(); self.m.pop()

Linked lists

A linked list is a chain of nodes. Each node holds a value and a pointer to the next. No indexes — to find item 4, you walk through 1, 2, 3 first. The trade-off: inserts are cheap if you already have a pointer.

No indexes insert O(1) with pointer search O(n) no shifting

Building and traversing
PYTHON
class Node:
    def __init__(self, val):
        self.val  = val
        self.next = None

a = Node(1); b = Node(2); c = Node(3)
a.next = b; b.next = c     # 1 → 2 → 3 → None

curr = a
while curr is not None:
    print(curr.val)
    curr = curr.next       # the core move: walk forward
1-4class Node — a node is just an object with two attributes: val (the data) and next (a reference to the next node, or None if it's the tail).
6-7a.next = b — this is how you "link" nodes. a.next stores a reference (memory address) to the object b. Now following a.next takes you to b.
9curr = a — start with a pointer at the head. We never change the head variable itself — we use curr to walk forward.
10while curr is not None — the loop runs as long as we haven't fallen off the end. The last node's .next is None, which ends the loop.
12curr = curr.next — the fundamental move. This is how you advance. Without this line you'd loop forever on the same node.
Reversal — animated
Reversing flips every arrow. We use three pointers: prev (starts at None), curr (starts at head), nxt (saves the next before we overwrite it).
1→2→3→4→None. Goal: 4→3→2→1→None.
Fast and slow pointers
PYTHON
# Find middle (tortoise & hare)
def middle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow

# Detect cycle (Floyd's algorithm)
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast: return True
    return False
3slow = fast = head — both pointers start at the same place. Slow moves 1 step per iteration; fast moves 2.
4fast and fast.next — we check both because fast moves 2 steps. If the list has odd length, fast lands on the last node (fast.next=None). Even length, fast goes to None directly.
7return slow — when fast reaches the end, slow is exactly at the middle. This works because fast covers twice the distance.
15if slow == fast — if there's a cycle, fast will lap slow and they'll meet inside the loop. In a finite list with no cycle, fast reaches None and they never meet.

Reverse linked listEasy

Reverse in-place with three pointers. Classic — everyone should be able to do this in their sleep.

1→2→3→4→None → 4→3→2→1→None
LeetCode ↗
prev=None, curr=head. While curr: nxt=curr.next; curr.next=prev; prev=curr; curr=nxt. Return prev.
def reverse(head):
    prev = None; curr = head
    while curr:
        nxt = curr.next
        curr.next = prev
        prev = curr; curr = nxt
    return prev

def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next; fast = fast.next.next
        if slow == fast: return True
    return False

Merge two sorted lists into one. Use a dummy head. Compare heads, attach smaller, advance that pointer.

def merge(l1, l2):
    dummy = curr = Node(0)
    while l1 and l2:
        if l1.val <= l2.val:
            curr.next = l1; l1 = l1.next
        else:
            curr.next = l2; l2 = l2.next
        curr = curr.next
    curr.next = l1 or l2
    return dummy.next
